# V17.1 Swing Generalization Fixes

**Goal:** Repair the V17.1 code paths that weaken Swing supervision and make validation/checkpoint selection unreliable, while keeping normal inference vision-only and preserving earlier model interfaces.

**Architecture:** V17.1 keeps the dual-modality V17 decoder, but uses the independent four-head mask path, temporally mixes each modality before mask generation, and adds training-only periodic pose and Swing-velocity heads. The main action output remains four sin/cos pairs and inference still ignores `qpos`.

## Tasks

1. Add failing regression tests for the V17.1 mask branch, auxiliary output shapes, temporal-mixer ordering, weighted joint loss, EMA updates, and checkpoint best-metric persistence.
2. Route `v17.1` through all V17 independent-mask initialization, forward, and decoder branches.
3. Move V17.1 temporal mixing before per-modality mask generation and avoid applying it twice to the fused grid.
4. Change V17.1 pose auxiliary supervision to four sin/cos pairs and add an independent Swing velocity sin/cos head.
5. Weight the main Swing prediction loss, correct circular R2 to use wrapped squared deviations, update EMA after every optimizer step, validate/save EMA weights, and restore best loss/Swing R2 on resume.
6. Run focused and compatibility tests, compile the changed Python files, and inspect the final diff before merging to `main`.
